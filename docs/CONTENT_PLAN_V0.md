# Lil8TD Content Plan v0

## Purpose

This document defines the initial playable content shape for Lil8TD MVP.
The goal is not content abundance.
The goal is a tight roster with clear interaction value.

## Content philosophy

MVP content should be:
- easy to read
- strategically distinct
- small enough to balance
- broad enough to create adaptation

Each content piece should justify itself through gameplay function.
If two pieces create almost the same decision, one of them probably should not exist yet.

## MVP content targets

### Towers
Recommended target:
- 6 core towers
- optional 1 or 2 later additions only if the interaction matrix really needs them

### Enemies
Recommended target:
- 8 core enemy types

### Loadouts
Recommended target:
- 3 compact loadout packages

### Maps
Recommended target:
- 1 primary MVP map
- optional 1 mirror-variant only if needed for testing, not product framing

## Tower roster proposal

## 1. Arrow Tower
Role:
- baseline cheap single-target DPS

Use:
- early defense
- reliable filler coverage
- upgrade branch baseline

## 2. Burst Tower
Role:
- anti-swarm / splash coverage

Use:
- punishes clustered light units
- lane cleanup support

## 3. Piercer Tower
Role:
- anti-heavy sustained lane pressure

Use:
- counters tanky units
- slower but efficient versus brutes

## 4. Frost Tower
Role:
- control / slow / timing extension

Use:
- boosts kill-zone value
- enables greedy damage setups

## 5. Support Beacon
Role:
- buff / aura / efficiency amplifier

Use:
- boosts nearby towers
- encourages intentional layout decisions

## 6. High-Cost Spike Tower
Role:
- expensive power anchor

Use:
- high commitment late-game stabilizer
- should not dominate the board alone

## Tower roster interaction goals

The roster should allow these decisions:
- pure cheap spam versus early stability
- anti-swarm versus anti-heavy specialization
- raw damage versus control-first setups
- isolated power spikes versus synergy builds

## Enemy roster proposal

## 1. Runner
- cheap, fast, fragile
- tests placement gaps and reaction speed

## 2. Grunt
- standard baseline enemy
- anchor for wave readability

## 3. Brute
- high health, slower
- tests anti-heavy planning

## 4. Swarmling
- many small bodies
- punishes single-target overinvestment

## 5. Shielded Unit
- reduced damage from a profile or front-facing mitigation depending on final design
- forces build diversity

## 6. Sprinter
- leak threat with burst speed
- punishes thin lane coverage

## 7. Support Carrier
- buffs nearby enemies or reduces defense efficiency indirectly
- creates target priority moments

## 8. Splitter
- splits on death into smaller units
- tests anti-swarm coverage and kill-zone planning

## Enemy interaction goals

The enemy roster should create pressure around:
- target mix
- timing windows
- lane control
- burst versus sustained damage
- control tower value

## Loadout package proposal

## 1. Balanced Core
- broad stable tower access
- no extreme bonuses
- best default teaching loadout

## 2. Tempo Control
- stronger control/support access
- weaker brute-force anchor options
- rewards positioning and efficiency

## 3. Pressure Forge
- stronger offensive pressure options or stronger economic conversion into pressure
- slightly riskier defensive baseline

## Loadout design rules

- loadouts should create style differences, not trap picks
- no loadout should depend on hidden advanced knowledge to be viable
- each loadout should support both solo and online play

## Wave progression guidance

MVP wave progression should teach the player gradually:
1. baseline lane defense
2. swarm response
3. heavy response
4. mixed composition adaptation
5. leak punishment and stabilization
6. pressure under escalation

## Content anti-goals

- do not ship 15 towers in MVP
- do not ship many enemies with only cosmetic differences
- do not create one obviously dominant universal tower
- do not create loadouts that hard-counter each other by draft alone
- do not overload the first map with gimmicks
